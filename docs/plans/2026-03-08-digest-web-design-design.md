# Digest Web Design Design

**Date:** 2026-03-08

## Goal

다이제스트 웹 페이지를 단순하고 직관적이면서 우아한 방향으로 개선한다. 우선순위는 가독성이며, 전체 톤은 한글 중심 서비스에 맞는 차분한 미니멀로 유지한다.

## Context

- 현재 사이트는 Jinja2 템플릿과 정적 CSS로 생성된다.
- `templates/base.html`, `templates/index.html`, `templates/archive.html`, `templates/daily.html`가 구조를 담당한다.
- `static/style.css`는 Pico CSS 위에 최소한의 보정만 추가한 상태다.
- 사용자는 최신 다이제스트를 빠르게 확인하고, 각 논문 요약을 긴 호흡으로 읽을 수 있어야 한다.

## Chosen Direction

세 가지 방향 중 `연구 노트보드`를 선택했다.

- 홈과 아카이브는 훑어보기 쉬운 구조를 우선한다.
- 상세 페이지는 각 논문을 또렷한 카드와 섹션 블록으로 정리한다.
- 장식보다 정보 계층과 읽기 흐름에 집중한다.

## Visual System

### Typography

- 본문 폰트: `Noto Sans KR`
- 제목 폰트: `Noto Serif KR`
- 본문 가독성을 우선하고, 제목에만 제한적으로 세리프를 사용해 차분한 인상을 만든다.
- 긴 문단을 고려해 line-height를 넉넉히 두고, 한 줄 길이를 제한한다.

### Color

- 배경은 약간 따뜻한 오프화이트를 사용한다.
- 본문 텍스트는 거의 검정에 가까운 잉크톤을 사용한다.
- 포인트 컬러는 낮은 채도의 청회색 또는 녹회색 계열로 제한한다.
- 시각적 구분은 그림자보다 경계선과 미세한 톤 차이로 만든다.

### Motion

- 기본 경험은 정적으로 유지한다.
- 필요한 경우 페이지 진입 시 약한 페이드 또는 위로 떠오르는 정도의 절제된 모션만 허용한다.

## Page Structure

### Home

- 상단에는 간결한 브랜드 헤더와 내비게이션을 유지한다.
- 헤더 아래에는 최신 다이제스트를 강조하는 `hero-summary` 블록을 배치한다.
- 최신 카드에는 날짜, 논문 수, 짧은 설명, 주요 CTA를 포함한다.
- 최근 기록은 테이블 대신 카드형 리스트 또는 단정한 리스트로 구성한다.

### Archive

- 기존 표 구조를 카드형 또는 넓은 리스트형 구조로 바꾼다.
- 날짜를 가장 먼저 읽히게 하고, 논문 수와 링크는 보조 정보로 둔다.
- 모바일에서도 자연스럽게 줄바꿈되는 block 기반 레이아웃을 사용한다.

### Daily Digest

- 상단에 날짜, 총 논문 수, 간단한 안내를 담은 `digest-header`를 둔다.
- 각 논문은 독립 카드로 구성한다.
- 제목, 저널, 소속 등 메타 정보를 먼저 묶고, `배경`, `방법`, `결과`, `의미`를 같은 규칙의 섹션 블록으로 배치한다.
- 키워드와 판정 근거는 별도 메타 영역으로 내려 읽기 흐름을 해치지 않게 한다.

## Implementation Scope

- `templates/base.html`에서 공통 래퍼, 폰트 로드, 헤더 구조를 정리한다.
- `templates/index.html`에서 최신 카드와 최근 기록 구성을 재설계한다.
- `templates/archive.html`에서 테이블을 탐색형 리스트 또는 카드 구조로 바꾼다.
- `templates/daily.html`에서 헤더, 논문 카드, 요약 섹션 블록, 메타 영역을 정리한다.
- `static/style.css`를 사이트 전용 스타일 시스템으로 재구성한다.
- 데이터 생성 로직은 유지하고, 템플릿에 필요한 최소한의 클래스만 추가한다.

## Testing Strategy

- `tests/test_site_generation.py`를 확장해 새 레이아웃의 핵심 클래스와 텍스트가 렌더링되는지 검증한다.
- 우선 `uv run pytest tests/test_site_generation.py`를 실행한다.
- 필요하면 `uv run python -m baiodigest.main --generate-only`로 실제 HTML 결과를 재생성해 눈으로 확인한다.

## Notes

- 한글 중심 서비스이므로 폰트 선택은 한글 가독성을 기준으로 결정했다.
- 전체 방향은 화려함보다 정돈된 정보 전달과 긴 읽기 경험에 초점을 둔다.
