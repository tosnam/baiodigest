# Archive Calendar Design

**Date:** 2026-03-08

## Goal

아카이브 페이지를 월간 캘린더 뷰로 전환한다. 각 날짜 칸에는 해당 일자 다이제스트의 논문 편수를 표시하고, 월 이동 네비게이션을 제공한다.

## Context

- 현재 아카이브는 모든 다이제스트를 세로 리스트로 보여준다.
- 사이트는 Jinja2 기반 정적 HTML 생성기로 빌드된다.
- GitHub Pages 같은 정적 호스팅을 유지해야 하므로 서버 라우팅 없이 동작해야 한다.
- 사용자는 아카이브에서 월 단위로 결과를 훑고, 특정 날짜 다이제스트가 있으면 바로 진입할 수 있어야 한다.

## Chosen Direction

세 가지 접근 중 `월별 아카이브 페이지 생성`을 선택했다.

- `/archive.html`은 최신 월 캘린더를 보여주는 진입점으로 유지한다.
- `/archive/YYYY-MM.html` 형식의 월별 정적 페이지를 추가 생성한다.
- 이전/다음 월 링크는 연속된 달 기준으로 항상 제공한다.
- 데이터가 없는 달도 범위 안에 있으면 `0편` 캘린더 페이지를 생성한다.

## URL Structure

- 진입점: `/archive.html`
- 월별 페이지: `/archive/YYYY-MM.html`
- 예시:
  - `/archive.html`
  - `/archive/2026-03.html`
  - `/archive/2026-04.html`

`/archive.html`은 최신 월과 같은 내용을 렌더링한다. 사용자가 처음 진입했을 때 가장 최근 결과를 바로 볼 수 있게 하기 위함이다.

## Calendar Behavior

### Monthly View

- 페이지당 한 달만 보여준다.
- 상단에는 월 제목과 이전/다음 월 버튼을 둔다.
- 캘린더는 7열 그리드로 구성한다.
- 주 시작 요일은 **일요일**이다.
- 요일 헤더는 `일 월 화 수 목 금 토` 순서를 사용한다.

### Day Cells

- 해당 월의 모든 날짜를 항상 표시한다.
- 달력 시작 위치를 맞추기 위해 앞뒤 빈 칸을 포함한다.
- 날짜 셀에는 최소한 날짜 숫자와 `논문 N편`을 표시한다.
- 다이제스트가 있는 날짜는 활성 상태로 보이고 일간 페이지 링크를 가진다.
- 다이제스트가 없는 날짜는 비활성 상태로 보이되 `0편`을 표시한다.

### Empty Months

- 생성 범위 안의 데이터 없는 달도 별도 월 페이지를 생성한다.
- 이 경우 모든 날짜 칸은 `0편`을 표시한다.
- 이전/다음 월 네비게이션은 그대로 유지한다.

## Rendering Model

생성기에는 월 단위 컨텍스트가 필요하다.

- `ArchiveMonthPage`
  - 현재 월 정보
  - 월 제목 문자열
  - 이전/다음 월 링크
  - 요일 헤더 목록
  - 캘린더 셀 목록
- `ArchiveDayCell`
  - 날짜
  - 현재 월 소속 여부
  - 논문 편수
  - 일간 페이지 링크 존재 여부
  - 활성/비활성 표시용 상태

월 범위는 가장 이른 다이제스트 월부터 가장 최근 다이제스트 월까지 계산한다. 그 사이의 빈 달도 포함한다.

## Styling Direction

- 기존 차분한 미니멀 톤을 유지한다.
- 월 제목은 큰 세리프 제목으로 처리한다.
- 이전/다음 버튼은 헤더의 pill 버튼 스타일을 재사용한다.
- 날짜 셀은 작은 카드처럼 구성한다.
- 다이제스트가 있는 날짜는 배경 또는 경계선 대비를 높인다.
- `0편` 날짜는 대비를 낮춘 비활성 톤으로 표현한다.
- 모바일에서도 7열 레이아웃은 유지하되 셀 내부 정보량을 최소화한다.

## Implementation Scope

- `src/baiodigest/generator/site.py`
  - 월 범위 계산
  - 일요일 시작 캘린더 셀 생성
  - 월별 아카이브 페이지 컨텍스트 생성
  - `/archive.html`과 `/archive/YYYY-MM.html` 렌더링
- `templates/archive.html`
  - 월간 캘린더 마크업으로 교체
- `static/style.css`
  - 아카이브 캘린더 관련 스타일 추가
- `tests/test_site_generation.py`
  - 월별 페이지 생성
  - 최신 월 진입점
  - 일요일 시작 요일 헤더
  - `0편` 표시
  - 이전/다음 월 링크
  - 활성 날짜 링크

## Testing Strategy

다음 시나리오를 자동 테스트로 고정한다.

- `2026-02`, `2026-03`, `2026-05`만 데이터가 있을 때 `2026-04` 페이지도 생성되는지
- `/archive.html`이 최신 월 캘린더를 렌더링하는지
- 요일 헤더가 `일 월 화 수 목 금 토` 순서인지
- 데이터 없는 날짜가 `0편`으로 표시되는지
- 데이터 있는 날짜만 일간 페이지 링크를 가지는지
- 이전/다음 월 링크가 연속 월 기준으로 생성되는지

## Notes

- 정적 사이트 제약을 유지하기 위해 쿼리스트링 기반 단일 페이지 방식 대신 월별 정적 페이지 방식을 사용한다.
- 빈 날짜와 빈 달도 모두 명시적으로 보이게 만들어, 수집 공백을 사용자가 직관적으로 파악할 수 있게 한다.
